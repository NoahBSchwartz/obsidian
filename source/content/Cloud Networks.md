Private: private cloud in its own data center
- Much more scalable, more secure, but requires significant infrastructure (and hard to rapidly deploy)
Public: shared environment hosted by provider
Options
1. Traditional IT: full stack
2. Infrastructure (as a service): half of stack (you manage middleware, data, runtime, applications) (ie. amazon web services, google cloud, IBM cloud)
3. Platform (as a service): 1/4 (you manage data and applications) (ie. google app engine, red hat)
4. Software (as a service): You don't manage anything (ie. JIRA, dropbox, docusign)
